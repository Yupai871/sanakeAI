

# 🐍 Snake AI — 工业级强化学习框架

基于 **Double DQN**、**优先经验回放（PER）**、**CNN 特征图状态空间** 与 **自动混合精度（AMP）** 构建的高性能贪吃蛇 AI。  
本项目不仅仅是一个入门 Demo，更融入了大量工业级强化学习的工程调优与防崩溃设计。

## 效果预览

智能体从完全随机探索起步，经过科学的奖励塑形与经验回放：
- **前 100 局**：学会避免撞墙，理解吃食物能获得高分。
- **约 1000 局**：掌握精准的寻路逻辑，路径规划趋于直线。
- **5000 局后**：探索率降至 1%，进入纯利用阶段，学会在蛇身极长时“S型绕行”以避免封死自己。

## 核心架构与重大优化

相比于传统的 DQN 教程，本版本进行了彻头彻尾的重构与优化：

1. **彻底废弃单步在线训练**：CNN 对时间高度相关的数据极易产生“灾难性遗忘”。现改为**纯 Batch 离线训练**（每走 4 步从 PER 池采样 256 条经验），梯度下降极其稳定。
2. **CNN 空间特征化 (7 通道)**：抛弃了让 CNN 强行理解数字大小的做法，将蛇的“上下左右”朝向转化为铺满全图的 One-hot Feature Map，完美契合卷积神经网络的局部平移不变性。
3. **动态网络全连接层尺寸**：`Conv_QNet` 内部使用 Dummy Tensor 动态推断尺寸，彻底消灭了硬编码的“魔法数字”。任意更改游戏分辨率或卷积核大小，代码都不会崩溃。
4. **SumTree NaN 免疫**：深度学习早期的异常梯度容易产生 `NaN`，一旦混入 PER 经验池会导致整棵树永久瘫痪。底层引入了严格的浮点数清洗截断机制。
5. **硬件自适应 AMP**：自动检测 CUDA 环境，按需启用 `torch.amp` 混合精度，显存减半，速度翻倍；在 CPU 环境下又自动透明回退，确保跨平台零报错。

```text
┌─────────────────────────────────────────────────────────────┐
│                        agent.py                             │
│  ├── get_state()      编码为 7×24×32 空间特征张量           │
│  ├── get_action()     ε-greedy 策略 (精确浮点衰减)          │
│  ├── remember()       只记不练，存入 PER                    │
│  └── train_long_memory()  每 4 步触发一次批量训练           │
└────────────┬──────────────────┬──────────────────┬──────────┘
             │                  │                  │
      ┌──────▼──────┐   ┌───────▼──────┐   ┌──────▼──────┐
      │   game.py   │   │   model.py   │   │  memory.py  │
      │ ├─ play_step│   │ ├─ Conv_QNet │   │ ├─ SumTree  │
      │ ├─ 曼哈顿   │   │ │ (动态感知) │   │ ├─ PER池    │
      │ │  奖励塑形 │   │ └─ QTrainer  │   │ └─ NaN保护  │
      └─────────────┘   └──────────────┘   └─────────────┘
```

## 状态空间设计 (State Space)

智能体不再读取单一的坐标数字，而是拥有“上帝视角”。游戏画面被严格编码为一个 **7 通道、24×32 的 3D 张量**：

```text
状态张量 shape: (7, 24, 32)

[空间实体通道] (稀疏二值图)
通道 0：食物位置     (食物所在坐标 = 1.0，其余 = 0.0)
通道 1：蛇头位置     (蛇头所在坐标 = 1.0，其余 = 0.0)
通道 2：蛇身位置     (蛇身所在坐标 = 1.0，其余 = 0.0)

[方向特征图通道] (全量填充，供 CNN 提取全局状态)
通道 3：当前方向 UP      (若向上，全通道填充 1.0，否则全 0)
通道 4：当前方向 RIGHT   (若向右，全通道填充 1.0，否则全 0)
通道 5：当前方向 DOWN    (若向下，全通道填充 1.0，否则全 0)
通道 6：当前方向 LEFT    (若向左，全通道填充 1.0，否则全 0)
```

## 奖励函数设计 (Reward Shaping)

使用基于**曼哈顿距离**的密集奖励塑形。辅助信号幅度被严格限制，**绝对不会**发生“蛇通过反复横跳刷分”的 Reward Hacking 现象。

| 事件 | 奖励值 | 说明 |
|------|--------|------|
| **吃到食物** | **+10** | 核心正反馈信号 |
| **死亡/超时** | **-10** | 撞墙、撞自己或绕圈超时的死亡惩罚 |
| 靠近食物 | +0.02 | 距离缩短，提供微小的方向引导 |
| 远离食物 | -0.02 | 距离增加，提供微小的偏离惩罚 |
| 距离不变 | 0 | 保持中性 |

## 算法原理

1. **Double DQN**: 
   将“动作选择”与“动作估值”解耦。Online 网络负责选出最佳动作，Target 网络负责评估该动作的 Q 值，有效消除 Q 值过估计（Overestimation）问题。
2. **Soft Target Update**: 
   抛弃传统的硬拷贝，使用 $\tau = 0.005$ 的 Polyak Averaging 缓慢更新 Target 网络，使训练目标像流水一样平滑。
3. **优先经验回放 (PER)**: 
   基于线段树 (SumTree) 实现 $O(\log N)$ 级别的高效采样。根据 TD-Error 赋予每条经验优先级，使得网络“更多地复习那些它预测错的场景”。并引入 $\beta$ 退火与 IS Weights 修正采样偏差。

## 快速开始

### 1. 环境要求
**Python 3.9+**，已全面适配 PyTorch 2.x 标准（包含 `weights_only=True` 安全加载）。
```bash
pip install torch pygame numpy tensorboard
```

### 2. 开始训练
```bash
python agent.py
```
- 前 5000 局为探索期，$\epsilon$ 从 100% 线性精确衰减至 1%。
- 每 50 局自动渲染一次画面（或当分数 > 30 时自动开启渲染）。
- 无需画面时游戏以极限速度在后台推演。

### 3. 断点续训与安全保存
你可以随时按下 `Ctrl+C` 中断训练，程序会捕获中断信号并**安全保存最后的训练现场**。
下次直接运行 `python agent.py` 即可无缝恢复进度、步数与优化器状态。

## 文件详解

- **`game.py`**: 环境核心。实现了无死锁的迭代式食物生成机制；包含防止无效绕圈的 Timeout 惩罚；将外部的固定方向输入转换为局部的“相对转向（直走/左转/右转）”。
- **`agent.py`**: 智能体逻辑。实现了 7 通道张量编码；接管了全局训练 Loop；掌控 Checkpoint 的存取。
- **`memory.py`**: 底层数据结构。手写高效率 SumTree；实现带自动纠错、防 NaN 污染的 PrioritizedReplayBuffer。
- **`model.py`**: 深度学习网络。支持动态尺寸推断的 `Conv_QNet`；集成了 AMP 混合精度与 Double DQN 逻辑的 `QTrainer`；纯张量 Gather 操作提速。

## 进阶扩展方向

目前的基础架构已非常坚固，若想进一步挑战极限分数，可尝试以下方向：

1. **Dueling DQN**: 
   修改 `Conv_QNet` 的全连接层，将输出拆分为状态价值 $V(s)$ 与动作优势 $A(s,a)$，最后再相加。这能让模型更好地理解“在某些危险状态下，无论采取什么动作都不好”。
2. **N-Step 奖励引导**: 
   在经验池入库前，累积未来 $N$ 步的衰减奖励。这能极大加速早期稀疏奖励的传播速度。
3. **Noisy Networks**: 
   移除 $\epsilon$-greedy 的硬编码探索，在网络线性层中加入可学习的参数化噪声，实现环境自适应的智能探索。
